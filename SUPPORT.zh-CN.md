# Support

## 1. 使用建议

根据问题类型选择渠道：
- 使用问题 / 接入咨询建议：GitHub Discussions（）
- 缺陷报告：GitHub Issues
- 安全漏洞：见`SECURITY.md`，请通过 GitHub PVR 私下报告
- 社区交流（非敏感）：Discord `https://discord.gg/e4nDukFAc6`

## 2.提问前检查

请先确认：
- 已阅读 `README.md` 与 `docs/` 对应章节
- 已搜索现有问题/讨论
- 能够提供最小的复现信息（输入、期望、实际）

## 3.最小问题模板

建议包含：
- 背景与目标
- 当前行为
- 期望的行为
- 复现步骤
- 相关或截图

## 4. 响应说明

这是一个早期项目，维护者会尽力及时响应，但不保证实时支持。

## 5. 通道边界

- 架构决策、需求变更、代码评审：必须回到GitHub（Issue/PR）。
- Discord用于实时讨论、答疑和协作组织，不作为最终决策记录。
- 安全问题不要在 Discord 或公开 Issue 披露细节。
