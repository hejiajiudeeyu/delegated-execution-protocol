# 远程子代理协议适用范围规范（v0.1）

状态：草案  
更新时间：2026-03-03

## 1. 规范目标

本规范用于开源哪些任务适合破解远程子代理协议体系承接，哪些任务不应进入当前协议参考范围，以保证：
- 协议对象可标准化（可描述、可签约、可验收）
- 协议涉及方风险可控（资源约束、超时、权限边界明确）
- 协议实现可持续发展（成功率、延迟、合规率可量化）

## 2. 准入原则（五问义务）

一个任务要进入 Remote Subagent Protocol 的参考范围，默认应同时满足以下 5 个条件：

1. 可合同化：输入、输出、合同可执行 JSON 描述。  
2.可异步化：任务异步执行，不依赖异步级实时交互。  
3.可权力等化：可基于`request_id`重试，不产生不负责任的后果。  
4. 可验收审核：结果可做签名校验、Schema 校验或规则校验。  
5、可控风险：资源上限、超时上限、故障退路、人工兜底可定义。  

任一条件不满足，应不默认纳入当前范围，或先进入实验白名单。

## 3. 典型任务分类与示例（MVP 优先）

| 分类 | 为什么适合远程子代理 | 典型示例 | 最小化交付件 |
| :--- | :--- | :--- | :--- |
| 文档与工程格式 | 本地一次跑通成本高，速度细节依赖专用工具链与经验 | `Markdown -> Word(.docx)`；`PDF -> Markdown`（表格/公式/标题体系） | 还原转换产物+质量报告（率、失败页、警告项） |
| 内容渠道化生产| 同源内容到多平台素材的拆分、改写、生成托盘通常需要大量| `Markdown -> 小红书图文搭配`（封面图、分卡文案、标签建议） | 图片包 + 对应文案 + 发布元数据 |
| 专业领域布局与诊断| 需要领域规则、库房和排障流程，不是单次提示即可稳定经验完成 | 分子实验问题排查（失败原因排序、复现实验建议、风险提示） | 诊断结论 + 证据链 + 下一步操作清单 |
| 基础设施密集型任务 | 需要新一代模型、初创服务、GPU/HPC 或复杂依赖环境 | 高成本OCR/解析模拟、专用模型推理、内部知识库编排 | 结果包 + 运行摘要（运行、资源、关键日志） |
| 标准化批处理任务 | 任务可批量化、指标化，适合做稳定质量基线与长期落地 | 批量液体抽取、文本分类打标、质量分析 | 成型结果 + 置信度 + 异常清单 |

## 4.当前不适合（红线）

以下任务不宜作为远程子代理协议常规收录对象：

1.强实时交互任务（如低延迟队列协作、实时控制闭环）。  
2. 访问买家本地运行时、内网资源或本地必须控制的任务。  
3.风险高不可逆动作（直接支付、生产编写操作、法律/医疗最终裁决）。  
4.无法抓取、仅靠偏好判断质量结果的任务。  

## 5.远景范围（v0.2+ 探索）

在机制完善后，Remote Subagent可逐步划分以下方向：

1.远程真实世界实验：涉及远程设备、实验台、线路下采样与结果回传。  
2. 人参与任务：Human-in-the-loop审核、复核、执行节点。  
3.高塔基础设施任务：专用仪器、企业氧气系统、长时作业预设。  
4.经验积累型任务：依赖长期案例沉淀与策略专家复用。  

## 六、上架治理建议

1.参考实现中的目录收录，应附有任务分类、风险等级、验收规则与失败退路。  
2.协议参考实现的审核，应使用 `docs/current/testing/subagent-admission-checklist.md` 统一审核。  
3. 对于“远景范围”任务，默认进入白名单或灰度，不直接全量开放。
